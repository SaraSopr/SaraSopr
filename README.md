## <img src="https://media.giphy.com/media/QssGEmpkyEOhBCb7e1/giphy.gif" width="28"> Featured Projects

<details>
<summary><b>LLMIntent — LLM-Driven SDN Traffic Controller</b></summary>

<br/>

> A software-defined networking control system where an LLM acts as the forwarding decision engine — replacing rule-based logic with context-aware reasoning.

| Stack | Scale | Performance | Security | Impact | Repository |
| :--- | :--- | :--- | :--- | :--- | :--- |
| Python, OpenAI API, RYU, OpenFlow 1.3, Mininet, Streamlit | Full network topology in-context | Incremental state deltas minimise token usage | Structured JSON contract, no rule-based fallback | LLM-native SDN control with anomaly remediation | [github.com/SaraSopr/LLMIntent](https://github.com/SaraSopr/LLMIntent) |

Designed a network controller where, at startup, the full topology is injected into the model context; subsequent requests transmit only **incremental state deltas**, minimising token usage while preserving coherent situational awareness. The LLM returns structured JSON routing decisions (next-hop, slice assignment) that RYU translates directly into OpenFlow rules. Includes **anomaly detection**, **automated remediation**, and a real-time monitoring dashboard built in Streamlit.

</details>

<details>
<summary><b>Raw Material Delivery Forecasting — Append Consulting × Hydro ASA</b></summary>

<br/>

> An industrial-scale forecasting system for raw material deliveries in aluminium production, developed at NTNU within the *Modern Machine Learning in Practice* course.

| Stack | Scale | Performance | Security | Impact | Repository |
| :--- | :--- | :--- | :--- | :--- | :--- |
| Python, XGBoost, CatBoost, scikit-learn, pandas | 20+ years of historical delivery & PO data | Asymmetric quantile loss (q=0.2) for risk control | Internal dataset, NDA-bound | **Top 20% / 100 competing teams** | Kaggle-style competition (NTNU, 2025) |

In a team of 3, engineered a cumulative delivery forecasting system for an industrial aluminium production context. Built features from **20+ years of historical delivery and purchase order data** — lag variables, rolling statistics, and seasonal patterns. Trained and benchmarked **XGBoost** and **CatBoost** models, optimising with **asymmetric quantile loss** to minimise overestimation risk in supply chain planning.

</details>

<details>
<summary><b>Travel Recommendation App — Master's Thesis (in progress)</b></summary>

<br/>

> A full-stack travel itinerary system driven by a multi-stage AI pipeline that semantically scores and clusters points of interest for personalised day-by-day plans.

| Stack | Scale | Performance | Security | Impact | Repository |
| :--- | :--- | :--- | :--- | :--- | :--- |
| FastAPI (async), SQLAlchemy 2, PostgreSQL, React + TS, Tailwind | Google Places API ingestion | Per-day K-Means clustering for proximity | JWT authentication | Master's thesis @ University of Trento | Private (in development) |

Designed and is building an ingestion pipeline that fetches POI data from the **Google Places API** and uses an LLM to enrich each POI with a **semantic feature vector** across thematic axes (family-friendliness, adventure level, etc.). At query time, POIs are scored against the user's travel profile (trip type, group composition, duration) and top candidates are clustered via **K-Means** into per-day geographically proximate segments. Backend: async **FastAPI + SQLAlchemy 2 + PostgreSQL + JWT**. Frontend: **React + TypeScript + Tailwind CSS**, deployed on Vercel.

</details>

<details>
<summary><b>PolyMind — Multi-Agent Conversational System with RAG</b></summary>

<br/>

> An autonomous multi-agent platform where each agent embodies a real historical figure, maintaining persona consistency across open-ended dialogue via dedicated RAG knowledge bases.

| Stack | Scale | Performance | Security | Impact | Repository |
| :--- | :--- | :--- | :--- | :--- | :--- |
| Python, FastAPI, React, LangChain, ChromaDB, OpenAI API | One ChromaDB per persona agent | Topic-agent-driven orchestration | API-keyed access | Research-grade conversational AI prototype | [github.com/SaraSopr](https://github.com/SaraSopr) |

Built an autonomous multi-agent system in which each agent **simulates a real historical figure** and **maintains its personality** across open-ended dialogue. A dedicated **topic agent** drives the conversation, while each persona agent queries its own **ChromaDB knowledge base** via RAG to respond strictly in character. Built with Python, FastAPI, React, LangChain, ChromaDB, and the OpenAI API.

</details>
